# Team Onboarding Notes

New engineers should spend their first week reading the service overview, setting up the local development environment, and pairing with a teammate on one small ticket. The goal is to understand how changes move from a local branch to review and deployment.

Code reviews should focus on correctness, maintainability, and whether the change matches the product requirement. Reviewers are expected to cite specific files or behaviors when requesting changes.

Architecture decisions are recorded in short markdown notes. Each decision note should include the context, the decision, and the tradeoffs that were considered.
